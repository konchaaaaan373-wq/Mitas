# Mitas for Alliance デモシナリオ

本書は、Mitas for Alliance を **地域医療連携推進法人・病院グループ・病院幹部・行政・ベンチャーパートナー** にお見せするためのデモストーリーです。

機能網羅型のデモではなく、**「なぜこの製品が必要か」 → 「どう使うか」 → 「現場にどう効くか」** の順に伝える構成にしています。

---

## 目的

「地域医療連携法人内で発生する人材不足を、施設横断で可視化し、Necoが安全に配置調整するための **地域医療人材OS**」が、実際にどのように動くかを、**5〜15分** で体験してもらう。

デモ中に強調するメッセージは以下です。

1. **可視化**：個別施設の採用難ではなく、連携法人全体の人材需給として捉え直す
2. **施設横断配置**：応援勤務・非常勤勤務・当直・オンコール・専門外来支援を施設横断で調整
3. **Necoが安全に運用**：要件確認・候補者打診・施設提案・勤務確定・稼働確認・請求まで一気通貫
4. **AIは将来の補助機能**：自律的なマッチングではなく、Neco担当者と施設管理者の意思決定を補助

## 強調するメッセージ

### マクロ
- 個別施設の採用課題ではなく、**連携法人全体の人材需給課題** として可視化する
- スキマバイト型ではなく、**信頼できる医療者を信頼できる医療機関に配置する**
- 介在する Neco が **要件・資格・経験・希望条件** を踏まえて安全に運用

### ミクロ
- 勤務枠登録時の **構造化入力補助**（想定患者数 / 業務範囲 / バックアップ体制 / 電子カルテ / 集合場所・持ち物 / 注意事項）→ 医療者へ打診しやすくなる
- proposal_status を**手で運べる5アクション**（打診済み／医療者承諾／医療者辞退／施設へ提案／取下げ）
- 勤務確定モーダルで **不足項目を事前にブロック**（時給未設定／終了時刻未設定 等）
- アプリ内リマインドで **抜け漏れを画面内で防ぐ**
- 滞留アラートで **24時間超／48時間超** を可視化
- AIは **「将来実装予定」** と明示し、最終判断は人間

## 想定質問と回答

### Q1. AIで配置を自動化するのか？
**A.** いいえ。本フェーズでは AI は要件整理／候補者推薦補助／リスク検知／提案文作成／需給レポート作成の **補助機能** として今後実装します。**最終判断は Neco担当者と施設管理者** が行います。連携法人医療では責任範囲・トラブル時の説明責任が重要なため、自律マッチングは最初から採用しません。

### Q2. 既存の人材紹介と何が違うか？
**A.** 単発の紹介ではなく、勤務枠登録 → 要件確認 → 配置候補 → 提案 → 承認 → 勤務確定 → 稼働 → 完了確認 → 請求 → 監査ログ までを **同じ画面でつなぐプラットフォーム** です。連携法人の経営層が **施設横断で需給を見られる** ところが最大の差別化です。

### Q3. RLSやプライバシーは大丈夫か？
**A.** Supabase の Row Level Security でロール別にアクセス範囲を制限しています。worker は自分のアサイン・プロフィールのみ、facility_admin は自組織のリソースのみ、alliance_admin は閲覧主体、neco_admin は配置運用者として全体を扱います。`docs/RLS_CHECKLIST.md` に詳細チェックリストを掲載しています。

### Q4. 連携法人内の他施設へ応援に行く場合、雇用関係はどうなる？
**A.** 既存の応援勤務契約スキームに準拠します。Mitas は契約締結を強制しません。勤務枠の登録から請求までを一気通貫で扱える基盤を提供する位置づけです。

### Q5. 単独病院やクリニック、訪問看護にも対応する？
**A.** 順次展開予定です。本フェーズは **地域医療連携法人版を最も深く磨く** ことに集中しています。単独施設も同じ proposal / assignment フローで運用できるため、画面文言を切り替えるだけで適用可能です。

### Q6. データの所有権は？
**A.** 各施設・連携法人が登録したデータは、それぞれの組織の所有です。Mitas は配置・運用・分析のための処理者として扱います。

### Q7. オンプレ／専用クラウドの要件はあるか？
**A.** 現在は Supabase + Netlify のクラウド構成です。オンプレ／専用環境のニーズがあればロードマップに反映します。

### Q8. 導入までのリードタイムは？
**A.** 連携法人の規模・既存契約スキームによりますが、初回ヒアリングから **デモ環境提供：2週間 / 本番稼働開始：1〜2ヶ月** を目安にしています。

### Q9. 既存の紹介会社や派遣会社との違いは？
**A.** 紹介・派遣は単発取引です。Mitas は **連携法人内の継続的な人材需給を可視化し、繰り返し起こる人材不足を計画的に充足する基盤** です。

### Q10. 価格モデルは？
**A.** 連携法人の規模・施設数・利用ロール数に応じた月額モデルを基本とします。詳細は別途見積もりです。
